## How I Used AI

I used AI tools primarily as:
- a debugging assistant,
- a React/Next.js learning companion,
- and for brainstorming audit recommendation logic.

I did not use one-shot code generation for the entire project. Instead, I implemented the application incrementally and verified each feature manually before moving forward.

AI assistance was most useful for:
- understanding Next.js App Router concepts,
- learning React state management patterns,
- structuring pricing research,
- and improving UI organization.

## Manual Verification

I manually verified:
- form persistence behavior,
- dynamic plan rendering,
- audit recommendation conditions,
- routing between pages,
- and pricing accuracy against official vendor pricing pages.

I adjusted or rejected several AI-generated suggestions that did not align with the assignment requirements or current pricing structures.

## Limitations Encountered

AI-generated suggestions occasionally:
- oversimplified API pricing models,
- proposed unrealistic enterprise pricing assumptions,
- or suggested over-engineered implementations too early in development.

I had to simplify certain approaches to keep the project maintainable and aligned with the assignment scope.

## What I Learned

This project significantly improved my understanding of:
- React component architecture,
- dynamic form rendering,
- state persistence with localStorage,
- Next.js routing,
- and structuring business logic separately from UI code.

It also taught me the importance of validating AI-generated output rather than accepting suggestions blindly.
